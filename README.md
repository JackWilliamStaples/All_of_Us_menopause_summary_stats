## BigQuery export pattern

Both scripts use the same general BigQuery export pattern:

1. Define a SQL query.
2. Run the query against `Sys.getenv("WORKSPACE_CDR")`.
3. Export the result to Google Cloud Storage using `bq_table_save()`.
4. Read the exported CSV shards back into R using `gsutil cat`.

## Shared analysis concepts

Both workflows generate descriptive summaries (limited to All of Us participants with female sex at birth in the final manuscript) from participant-level indicators for:

- Short-read whole genome sequencing availability (`srWGS`)
- Electronic Health Record (`EHR`) data availability
- Menopause-related EHR condition codes
- Menopause-related survey responses
- Demographic and sociodemographic variables
- Genetic ancestry group labels
- Age and menopause-transition age groupings

Common menopause-related sources include EHR menopause diagnostic codes and the survey question:

```text
Have your menstrual periods stopped permanently?
```

The scripts separately consider participants who answered “yes” and participants with any answer to this question. “Yes” responses include “Yes, none” and “Yes, but hormone.” Any-answer responses include periods not stopped, yes responses, not sure, prefer not to answer, and skip responses.

### Female-sex filtering

The calculated menopause descriptive summaries in both workflows for the manuscript are filtered to participants with sex at birth recorded as Female.
---

## Sample size reporting

The workflows apply All of Us Research Program small sample size compliant reporting conventions.

In general:

- Counts less than or equal to 20 are rounded to 20 or represented as suppressed small samples sizes ("N ≤ 20"" or "20").
- Counts greater than 20 are rounded up to the nearest multiple of 5.
- Some computed tables may have small sample size labels as `less_than_equal_20` in the N column.

Treat these values as suppressed/disguised small sample size counts.

- Select useful EHR condition fields.
- Construct condition identifiers from concept name, concept code, and vocabulary.
- Keep the earliest condition date per person-condition pair.
- One-hot encode menopause-related EHR conditions.
- Create matching date columns for earliest condition dates.

## Reproducibility notes

- Confirm that the workspace is using the intended All of Us Controlled Tier dataset version before running each script.
- Version 8 outputs should be generated from Controlled Tier Dataset v8.
- Version 7 outputs should be generated from Controlled Tier Dataset v7.
- Files ending in `_V8_freeze09182025.tsv` or `_V7_freeze09182025.tsv` should be treated as frozen analysis files for reproducibility.
- Several frozen files are read back into the workflows rather than regenerated in every run from the original SQL query of the Controlled Data Repository.
- The primary data used for analyses and results summaries for the manuscript were limited to participants with female sex at birth.
